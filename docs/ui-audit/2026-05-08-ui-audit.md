### Top 3 Blockers

1. **Stripe Elements is a placeholder, not a real form.** [checkout/page.tsx:412-422](apps/web-v2/src/app/(app)/checkout/page.tsx#L412-L422) renders dashed-border divs labeled "Stripe Elements — Card Number (placeholder)". 3. **No automated a11y or working linter.** No `@axe-core/cli`, `axe-playwright`, `pa11y`, or `lighthouse` in any package.json. Playwright's `compliance` project tests Canadian regulatory facts ([e2e/compliance/canadian-scrap.spec.ts](apps/web-v2/e2e/compliance/canadian-scrap.spec.ts)), **not WCAG**. `pnpm lint` in `apps/web-v2` blocks on an interactive eslint setup prompt — **lint never runs**. AODA exposure is unmeasured.

## Critical Issues (P0) — Launch-Blocking

### 1. Checkout payment form is a placeholder
**Location:** [checkout/page.tsx:412-422](apps/web-v2/src/app/(app)/checkout/page.tsx#L412-L422)
**Severity:** CRITICAL — functional, not UI
**Affected users:** Every buyer. 100% of checkout flow.
**Risk:** No revenue capture; orders create with zero card validation; PCI scope unclear.

```tsx
<div className="h-10 rounded border-2 border-dashed border-night-600 bg-night-850 flex items-center justify-center text-xs text-night-300">
  Stripe Elements — Card Number (placeholder)
</div>
```

**Fix:** Wire `@stripe/react-stripe-js` `<CardElement>` (the `stripe` SDK is already in deps) and gate the "Pay" button on `stripe.confirmCardPayment` success. Effort: 4–8h (mostly testing + 3DS).

### 2. Bill 96 commitment unfulfilled
**Location:** [src/lib/i18n.ts:1-18](apps/web-v2/src/lib/i18n.ts#L1-L18), 5 hardcoded-English page templates
**Severity:** CRITICAL — legal risk for QC market
**Affected users:** ~24% of TAM (Quebec).
**Risk:** Bill 96 fines + reputational harm if QC pilot launches.

The i18n helper exists and has FR-CA strings for `nav`/`dashboard`/`listings`/`auctions`/`errors` (~70 keys). But:
- No `[locale]` route segment.
- No `<NextIntlClientProvider>` wrapper.
- `<html lang="en">` is hardcoded in [app/layout.tsx:56](apps/web-v2/src/app/layout.tsx#L56).
- Every page string is hardcoded English literals.

**Recommended path:**
- **Short-term:** Add a visible `EN | FR` toggle in the header that calls `setLocale()` + reload, so the existing `t()` helper actually flows. Replace ~40 high-traffic strings (nav labels, dashboard title, primary CTAs, error banners) with `t()` calls. Update `<html lang>` dynamically.
- **Long-term (defer):** Full `[locale]` segment + native FR-CA review before QC marketing.
- **Don't do:** machine-translate to ship "FR-CA". The deferral comment in `i18n.ts` is correct — Quebec users will spot Google Translate immediately.

Effort short-term: 8–12h.

### 3. Cross-border switch keyboard-inaccessible (login)
**Location:** [(auth)/login/page.tsx:841-859](apps/web-v2/src/app/(auth)/login/page.tsx#L841-L859)
**Severity:** CRITICAL — WCAG 2.1.1 Keyboard fail
**AODA risk:** Up to $100K/day in Ontario for unremediated public-facing failures.

```tsx
<div role="switch" aria-checked={crossBorder}
     onClick={() => setCrossBorder((v) => !v)}
     className="...">
```

A `<div role="switch">` with no `tabIndex={0}` and no keyboard handler is invisible to keyboard users. The same pattern likely repeats — search for `role="switch"`.

**Fix:** Replace with `<button type="button" role="switch" aria-checked={...} onClick={...}>`. Native button gives focus + Space/Enter handling for free.

### 4. Disabled primary button — contrast fail
**Location:** [components/ui/shadcn/button.tsx:20](apps/web-v2/src/components/ui/shadcn/button.tsx#L20)
**Severity:** CRITICAL — WCAG 1.4.3 Contrast (~2:1 vs required 4.5:1)
**Affected users:** Anyone seeing a disabled CTA — disabled states currently render illegibly.

```tsx
"bg-brand-600 text-white ... disabled:bg-brand-300"
```

`brand-300` (#ffa83c) on white text is ~1.9:1. Disabled buttons are unreadable and invite click-through attempts.

**Fix:** `disabled:bg-night-700 disabled:text-night-400` (matches the secondary disabled treatment). Effort: 5 min.

### 5. Listing card uses `<img>` not `<Image>` — LCP regression
**Location:** [listings/page.tsx:213-220](apps/web-v2/src/app/(app)/listings/page.tsx#L213-L220) (eslint-disabled inline)
**Severity:** CRITICAL — perf for Mike on 4G + Field Estimator
**Risk:** Listing thumbnails are the LCP element on the listings grid; raw `<img>` blocks Next's automatic resizing/AVIF/WebP/lazy-loading. On 4G with 8 cards visible × ~500KB photos = unusable.

**Fix:** Replace with `<Image fill sizes="(max-width: 640px) 100vw, 25vw" />` and let Next handle the rest. Verify `images.remotePatterns` covers the thumbnail host.

### 6. ListingsTable invisible dividers
**Location:** [listings/ListingsTable.tsx:185](apps/web-v2/src/components/listings/ListingsTable.tsx#L185)
**Severity:** CRITICAL — table rows visually merge on dark
**Issue:** `divide-zinc-100` (very light gray) on `bg-night-850` produces ~1.05:1 contrast — dividers are invisible. Buyers comparing rows can't see row boundaries.
**Fix:** `divide-night-700/60`.

### 7. Auth token leaked to localStorage (XSS vector)
**Location:** Five files: [(app)/layout.tsx:103](apps/web-v2/src/app/(app)/layout.tsx#L103), [lib/api.ts](apps/web-v2/src/lib/api.ts), [dashboard/page.tsx:213](apps/web-v2/src/app/(app)/dashboard/page.tsx#L213), [chat/page.tsx](apps/web-v2/src/app/(app)/chat/page.tsx), [components/layout/MatexCopilot.tsx:114](apps/web-v2/src/components/layout/MatexCopilot.tsx#L114)
**Severity:** CRITICAL — security
**Risk:** Any XSS (a single unsanitized seller-supplied string anywhere) reads `localStorage.matex_token` and exfiltrates it. Stripe webhooks, listing descriptions, chat messages — all expansion surfaces.

The login flow already POSTs the token to `/api/auth/session` to set a cookie. That cookie is the right primary; the localStorage copy is the redundant attack surface.

**Fix:** Stop reading/writing `matex_token` in localStorage. Have `getUser()` fetch from a `/api/auth/me` endpoint that reads the httpOnly cookie. Move client-side gating off the token presence (use a `/api/auth/me` 200/401 response). Effort: 4–8h, gated by gateway-side support.

### 8. Hero "Continue with Google" / "LinkedIn" buttons are dead UI
**Location:** [(auth)/login/page.tsx:1363-1396](apps/web-v2/src/app/(auth)/login/page.tsx#L1363-L1396)
**Severity:** CRITICAL — trust-breaking on first impression
**Issue:** Both buttons render full-fidelity but have no `onClick`. Click → nothing happens. First-impression trust collapse.
**Fix:** Either remove until OAuth ships, or wire to a "coming soon" tooltip. Don't ship dead CTAs.

### 9. Password requirement copy mismatches code
**Location:** [(auth)/login/page.tsx:498, 970](apps/web-v2/src/app/(auth)/login/page.tsx#L498)
**Severity:** CRITICAL — users hit a silent rejection
**Code:** `if (password.length < 12) errs.password = "Password must be at least 12 characters."`
**Copy on field:** `placeholder="Min. 8 characters"`
**Fix:** Update placeholder + add a hint line. Effort: 1 min.

### 10. AI Advisor demo data leaking to production
**Location:** [auctions/[id]/page.tsx:455](apps/web-v2/src/app/(app)/auctions/[id]/page.tsx#L455)
**Severity:** CRITICAL — false advice = legal/financial liability
**Issue:** "consider bidding up to $31,000" is hardcoded into every auction room. There is no real model behind it.
**Fix:** Either gate behind a feature flag OR wire to `pricing.get_recommendation` (already exists per the architecture). If neither — remove. Effort: 30 min to remove.

## Design System Health

**Tokens defined:** `brand` (orange #e87722), `accent` (amber), `steel`, `surface`, `night` (12-step), `info` (blue), `success`, `warning`, `danger`. **Strong.**

**Tokens violated:** 40+ files mix raw Tailwind colors (`zinc-*`, `red-*`, `blue-*`, `emerald-*`, `amber-*`, `purple-*`, `orange-*`) into the codebase. Worst offender: [checkout/page.tsx](apps/web-v2/src/app/(app)/checkout/page.tsx) — ~30 instances of `text-blue-*` / `bg-blue-*` (no `blue-*` exists in tokens — Tailwind defaults render).

**Duplicated primitives:**
- Two button systems: legacy `.btn-primary` etc. in `globals.css:438` AND shadcn `<Button variant="primary">`. The shadcn variant uses `brand-600`, the global class uses `brand-500`. **Same brand, inconsistent shade.**
- Login has its own `GlassEmailInput`/`GlassPasswordInput`/`SubmitButton`/`ChipSelect` that duplicate what shadcn provides.
- `EmptyState` defines its own `CTAButton` rather than using `<Button>` ([EmptyState.tsx:35-55](apps/web-v2/src/components/ui/EmptyState.tsx#L35-L55)).

**Recommendation:** add a tailwind plugin or ESLint rule blocking `text-(zinc|red|blue|emerald|amber|purple|orange|yellow|sky|teal|rose)-` outside of `tailwind.config.js` itself. This is a one-shot policy that prevents future drift.
